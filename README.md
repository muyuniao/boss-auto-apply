# Boss 直聘自动投递油猴脚本

这是一个轻量版 Tampermonkey/篡改猴脚本，用于在 Boss 直聘职位列表页按筛选规则自动发起沟通/投递。

> 重要：脚本默认开启“试运行”，不会真实投递。真实投递前请确认筛选条件、投递间隔和平台规则。脚本不会登录账号、不会读取/保存你的密码，也不会绕过验证码或风控。

## 文件

- `boss-auto-apply.user.js`：可直接安装到 Tampermonkey 的用户脚本。
- `package.json`：仅用于本地语法检查。

## 安装

1. 安装 Tampermonkey/篡改猴。
2. 打开 Tampermonkey 管理面板，新建脚本。
3. 将 `boss-auto-apply.user.js` 全部内容复制进去并保存。
4. 在浏览器中自行登录 Boss 直聘。
5. 打开职位页，例如：
   - `https://www.zhipin.com/web/geek/job`
   - `https://www.zhipin.com/web/geek/jobs`
   - `https://www.zhipin.com/web/geek/job-recommend`

页面右下角会出现「Boss 自动投递助手」面板。

## 推荐使用流程

1. 保持默认「试运行」开启。
2. 设置职位/公司/详情关键词、薪资范围、单次上限和投递间隔。
3. 点击「保存」。
4. 点击「检测岗位数据」，打开浏览器控制台查看当前页岗位字段是否能被读取。
5. 点击「开始」试运行，确认日志里的“拟投”岗位符合预期。
6. 确认无误后，取消「试运行」，再次点击「开始」并在弹窗中确认。

## 功能

- 自动读取 Boss 职位卡片中的内部数据。
- 按职位名、公司名、岗位详情关键词过滤。
- 可选拉取岗位详情，用于详情关键词、BOSS 活跃时间和已沟通状态判断。
- 薪资范围过滤（支持常见 `15-25K`、`1-3万` 格式）。
- 本地今日投递历史，避免同日重复投递。
- 单次上限、本地日上限、随机投递间隔。
- 可选自动翻页/滚动加载更多。
- 默认试运行，不真实调用投递接口。
- 对 Boss 返回的 `开聊提醒` / `chatRemindDialog` 默认按成功记录，并在控制台输出原始响应，便于确认是否需要人工处理。

## 注意事项

- Boss 页面结构可能变化；如果检测不到 `securityId / encryptJobId / lid`，脚本会跳过该岗位。可点击「检测岗位数据」查看控制台输出。
- 真实投递调用的是 Boss 页面自身接口，需要你已经在浏览器中登录。
- 不要设置过短间隔或过高上限，避免触发平台限制。
- 脚本不包含自定义招呼语发送。当前轻量版只发起 Boss 默认沟通/投递流程。
- 如果出现验证码、账号异常、今日沟通上限等提示，请停止脚本并人工处理。
- 如果日志显示“已触发开聊提醒，按成功记录”，建议打开 Boss 的聊天列表确认是否已经产生会话；如没有，打开面板里的“调试日志”，再把控制台里 `[boss-auto-apply-lite] 投递接口响应` 的 `raw` 结构脱敏后发我继续适配。

## 本地校验

如果本机安装了 Node.js：

```bash
npm run check
```

该命令只做 JavaScript 语法检查，不会运行脚本。
